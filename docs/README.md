# QAgent 文档目录

本文档目录已经按版本和用途重新整理。根目录只保留当前系统使用所需的入口文档；历史设计、验收、报告和问题归档放入对应版本目录。

## 当前入口

| 文档 | 用途 |
| --- | --- |
| [USER_GUIDE.md](./USER_GUIDE.md) | 面向人类用户的当前系统使用手册 |
| [AGENT_GUIDE.md](./AGENT_GUIDE.md) | 面向 agent 的当前系统使用规范和研究流程 |
| [AGENT_RESEARCH_PATHS.md](./AGENT_RESEARCH_PATHS.md) | 面向 agent 的回测优化研究路径和 10 个行动方案 |

| --- | --- | --- |
| [v1.0/](./v1.0/) | 历史归档 | 早期 US equities 单市场研究系统的目标、需求、开发计划、验收和截图资产 |

## 维护规则

- 根目录文档描述当前可用系统，不再承载历史 backlog 或阶段验收记录。
- v1.0、v2.0 文档保留历史语境，里面出现的旧路径代表当时的文件位置。
- v3.0 是当前开发和后续迭代的主文档目录，进度更新优先写入 [v3.0/3.0-development-progress.md](./v3.0/3.0-development-progress.md)。
- 新的长期设计文档、迁移报告和验收报告应放入对应版本目录，不再散落在 `docs/` 根目录。
