# QAgent Next 功能补充记录

> 文档状态：开发过程补充需求记录  
> 创建日期：2026-05-17  
> 用途：记录开发过程中发现的可优化功能、缺失需求和后续可加入开发计划的事项。

## 使用规则

1. 发现新增功能、性能优化或需求缺口时，先记录到本文件。
2. 若该项对当前阶段目标必要，可以直接加入 `development-task-directory.md`。
3. 若不是当前阶段必要内容，保留为 OPEN，后续阶段评审时处理。
4. 实现完成后，将状态改为 DONE，并补充验收证据。

状态枚举：

- `OPEN`：已发现，尚未安排。
- `PLANNED`：已加入开发任务目录。
- `DOING`：正在实现。
- `DONE`：已完成并验收。
- `DROPPED`：明确不做，需要写明原因。
