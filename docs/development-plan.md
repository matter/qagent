# QAgent 开发计划

> 基于 requirements.md v2.3 拆解。每个 Phase 细分为里程碑（Milestone），每个里程碑有明确的交付物和验收标准。

---

## 总览

```
Phase 1: 项目骨架 + 数据基础        M1.1 → M1.2 → M1.3 → M1.4
Phase 2: 因子研究                    M2.1 → M2.2 → M2.3
Phase 3: 模型与策略                  M3.1 → M3.2 → M3.3 → M3.4
Phase 4: Agent + 信号 + 闭环         M4.1 → M4.2 → M4.3
```

---

## Phase 1：项目骨架 + 数据基础

> 目标：项目可运行、数据可获取、行情可浏览。

### M1.1 项目骨架

**交付物：**

- Python 项目结构（包、模块、入口）
- FastAPI 应用启动，返回 health check
- React 前端脚手架（Vite + TypeScript + Ant Design），dev proxy 到 FastAPI
- 配置管理（config.yaml 加载）
- 结构化日志（JSON，按日滚动）
- DuckDB 连接管理（单例，进程内共享）
- 统一任务执行框架骨架（任务创建、状态流转、运行记录、超时控制）

**目录结构：**

```
qagent/
├── backend/
│   ├── app.py                  # FastAPI 入口
│   ├── config.py               # 配置加载
│   ├── db.py                   # DuckDB 连接管理
│   ├── logger.py               # 日志
│   ├── tasks/                  # 统一任务执行框架
│   │   ├── executor.py         # 任务执行器（隔离、超时、重试）
│   │   ├── models.py           # 任务状态模型
│   │   └── store.py            # 任务运行记录存储
│   ├── services/               # 业务服务层（后续各模块填充）
│   ├── api/                    # HTTP API 路由（后续各模块填充）
│   └── providers/              # 数据源抽象
├── frontend/
│   ├── src/
│   │   ├── App.tsx
│   │   ├── layouts/            # Ant Design 侧边栏布局
│   │   ├── pages/              # 各页面（后续填充）
│   │   └── api/                # axios 封装
│   ├── vite.config.ts
│   └── package.json
├── data/                       # 运行时数据（git ignore）
├── config.yaml
└── pyproject.toml
```

**验收标准：**

- [ ] `python -m qagent` 启动 FastAPI，`/api/health` 返回 200
- [ ] `cd frontend && npm run dev` 启动前端，访问 localhost:5173 显示 Ant Design 布局框架
- [ ] 配置文件加载正常，日志输出到文件
- [ ] 任务框架可创建任务、记录状态（排队→运行→完成/失败/超时）、保存运行记录
- [ ] DuckDB 文件创建成功

### M1.2 数据获取与存储

**交付物：**

- DataProvider 抽象接口
- YFinanceProvider 实现（含防限流策略）
  - 股票列表：nasdaqtrader.com FTP 获取
  - 日线数据：yf.download 批量模式
  - 批量 500 只/批 + 随机延迟 + 指数退避 + requests-cache
  - 可中断续传（进度持久化到 update_progress.json）
- DuckDB 表结构
  - `stocks`：股票基础信息（ticker, name, exchange, sector, status）
  - `daily_bars`：日线 OHLCV + 复权因子（ticker, date, open, high, low, close, volume, adj_factor）
  - `index_bars`：指数/基准数据（SPY）
  - `data_update_log`：数据更新日志
- 交易日历集成（exchange_calendars，NYSE 日历）
- 数据质量校验（价格跳变、成交量异常、时间断裂）
- 增量更新逻辑（只请求本地缺失的日期区间）

**验收标准：**

- [ ] 全量初始化：能拉取 ≥100 只股票 × 1 年历史数据，写入 DuckDB
- [ ] 增量更新：二次执行时只请求缺失日期，请求量显著减少
- [ ] 中断续传：kill 进程后重启，从断点继续而非从头开始
- [ ] 限流：批次间有随机延迟，429 后自动退避，不崩溃
- [ ] 数据质量报告：更新后输出异常记录数
- [ ] 查询性能：单股 10 年日线查询秒级以内（PoC）

## Phase 2：因子研究

> 目标：因子可编写、可评价、可管理。

### M2.1 技术指标库 + 标签定义 + 因子协议

**交付物：**

- 技术指标适配层 `qagent.indicators`
  - TA-Lib 主引擎（RSI, MACD, ADX, ATR, OBV, MFI, BBands 等）
  - pandas-ta 补充引擎
  - 统一接口：输入 Pandas Series/DataFrame，输出 Pandas Series
  - 无 TA-Lib 时的降级方案（纯 pandas-ta 回退 + 安装提示）
- 标签定义管理
  - DuckDB 表：`label_definitions`（name, target_type, horizon, benchmark, status）
  - 预置标签：fwd_return_5d, fwd_return_20d, fwd_rank_5d, fwd_excess_5d
  - CRUD API：`/api/labels`
  - 标签值自动计算（给定行情数据和标签定义，生成 label 列）
- 因子协议 FactorBase
  - 基类定义（name, description, params, category, compute()）
  - 因子脚本加载机制（从文件系统加载 .py，实例化 FactorBase 子类）
  - 因子脚本执行隔离（独立线程 + 异常边界 + 超时控制）

**验收标准：**

- [ ] `ta.rsi(close, 14)` 返回正确的 RSI 值（与 TA-Lib 官方输出一致）
- [ ] 标签定义 CRUD 正常，预置标签可查看
- [ ] 编写一个最简因子脚本（如 20 日动量），通过 FactorBase 加载并执行
- [ ] 因子脚本抛异常时不崩溃系统，返回错误信息

---

### M2.2 因子计算引擎 + 评价体系

**交付物：**

- 因子计算引擎
  - 输入：因子脚本 + 股票池 + 日期范围
  - 输出：因子值 DataFrame（date × ticker）
  - 缓存：因子值按 (因子版本, 股票池, 日期范围) 缓存至 DuckDB
  - 缓存失效：因子代码更新时自动标记旧缓存失效
- 因子评价引擎
  - 输入：因子值 + 标签定义 + 股票池 + 日期范围
  - 输出：评价报告（IC/IR/IC 胜率/分组收益/换手率/覆盖率）
  - 评价结果保存至 DuckDB：`factor_eval_results`
  - 评价结果绑定：因子版本 + 标签定义 + 股票池 + 运行记录 ID
- 内置因子模板（~18 个预置因子 .py 文件）
- 通过任务框架执行（有运行记录、超时控制）

**验收标准：**

- [ ] 对 100 只股票 × 3 年数据计算 Momentum_20 因子，结果正确
- [ ] 因子评价绑定标签 fwd_return_5d，输出 IC 均值、IR、分组收益
- [ ] 修改因子代码后重新计算，旧缓存自动失效
- [ ] 全市场因子计算性能 PoC（5000 股 × 10 年，分钟级以内）

---

### M2.3 因子库管理 + 因子研究页面

**交付物：**

- 因子库管理
  - 保存因子：名称、描述、源码、参数、分类、状态（Draft）
  - 版本管理：同名因子修改后自增版本号，保留历史版本
  - 状态流转：Draft → Validated（有评价结果时可手动提升）
  - 引用保护：被特征集引用的因子不可删除
  - 浏览筛选：按分类/状态筛选，按 IR 排序
  - API：`/api/factors` CRUD + `/api/factors/{name}/evaluate`
- 前端页面：因子研究
  - 因子编辑器（Monaco Editor + 模板选择下拉）
  - 评价参数配置面板（标签定义选择、股票池选择、日期范围）
  - 评价报告展示（IC 时序图、分组收益图、统计表）
  - 因子库列表（含状态标签、IR 排序）

**验收标准：**

- [ ] 在 UI 中编写因子 → 选择标签和股票池 → 执行评价 → 查看报告
- [ ] 保存因子后在因子库列表中可见，显示状态和 IR
- [ ] 修改因子代码保存后版本号自增，旧版本可查看
- [ ] 因子脚本语法错误时展示完整 traceback，不影响系统
- [ ] 因子状态可从 Draft 提升为 Validated

**Phase 2 整体验收：** 可以编写因子、绑定标签评价、查看报告、管理因子库。

## Phase 3：模型与策略

> 目标：端到端研究链路打通（因子 → 特征 → 模型 → 策略 → 回测）。

### M3.1 特征集 + 预处理 + 回测 PoC

**交付物：**

- 特征集管理
  - 从因子库圈选因子 → 查看相关性矩阵 → 保存
  - 预处理配置（缺失值处理、去极值、标准化、中性化开关）
  - 预处理参数训练时保存、预测时复用
  - DuckDB 表：`feature_sets`、`feature_set_factors`
  - API：`/api/feature-sets`
- 回测引擎 PoC（不含 UI，纯后端验证）
  - T+1 开盘成交口径实现
  - 佣金 + 滑点成本模型
  - 多股票持仓更新（含停牌、退市处理）
  - 等权仓位管理
  - 输出：净值序列 + 基础指标（年化收益、最大回撤、夏普）

**验收标准：**

- [ ] 特征集可创建、关联因子版本、保存预处理配置
- [ ] 因子相关性矩阵计算正确，高相关对（>0.7）可识别
- [ ] 回测 PoC：给定固定信号，T+1 开盘成交，成本计算正确
- [ ] 回测 PoC：停牌股不成交，退市股自动清仓
- [ ] 回测 PoC：等权持仓 50 只 × 5 年，秒级到分钟级完成

---

### M3.2 模型训练

**交付物：**

- 模型接口 ModelBase（fit/predict/get_params/feature_importance）
- LightGBMModel 实现（回归 + 分类）
- 训练流程
  - 输入：特征集 + 标签定义 + 训练配置
  - 训练方式：单次划分 / 滚动窗口 / 扩展窗口
  - 时序验证：purge_gap 隔离期
  - 异步执行 + 进度反馈 + 超时控制
- 模型保存
  - 自包含目录：model.joblib + metadata.json
  - metadata 含：特征集 ID、特征名列表、预处理参数、模型参数、标签配置、评估指标
- 训练输出
  - 评估指标（IC, RMSE, R² / AUC, F1）
  - 特征重要性
- DuckDB 表：`models`（元数据）、文件系统存储模型文件
- API：`/api/models` + `/api/models/train`

**验收标准：**

- [ ] 基于特征集 + fwd_return_5d 标签完成一次时序训练
- [ ] 训练结果包含评估指标和特征重要性
- [ ] 保存模型 → 重新加载 → predict 输出与训练时一致
- [ ] 训练失败时不生成模型包，运行记录保留错误栈
- [ ] 模型训练超时可终止

---

### M3.3 策略 + 回测引擎

**交付物：**

- 策略协议 StrategyBase
  - generate_signals()：输入 context（行情、因子值、模型预测），输出信号
  - required_factors() / required_models()：声明依赖
- 仓位管理组件
  - 等权、信号加权、最大持仓约束（可组合）
- 完整回测引擎（基于 M3.1 PoC 扩展）
  - 自动编排管线：因子计算 → 预处理 → 模型预测 → 策略信号 → 仓位管理 → 回测
  - 完整指标输出（收益、风险、风险调整、交易指标）
  - 回测结果保存（含回测配置、策略版本、依赖链快照）
  - 结果分级标识（探索性/正式）
- DuckDB 表：`strategies`、`backtest_results`、`backtest_trades`
- API：`/api/strategies` + `/api/strategies/backtest`

**验收标准：**

- [ ] 编写策略引用因子和模型 → 回测 → 输出完整报告
- [ ] 管线自动编排：策略声明的因子和模型自动计算/预测
- [ ] 回测指标正确：夏普、最大回撤、换手率等
- [ ] 策略依赖的因子/模型为 Draft 时，结果标记为探索性

## Phase 4：Agent + 信号 + 闭环

> 目标：Agent 可访问系统，信号可生成，系统闭环。

### M4.1 MCP Server

**交付物：**

- MCP Server 实现
  - 基于 FastMCP，SSE 传输，`app.mount("/mcp", ...)`
  - 工具注册：数据查询、因子评价、模型训练、回测、信号生成、分组管理、任务状态查询
  - 长任务异步模式：返回 job_id + get_job_status 轮询
- MCP 工具调用经过服务层（不绕过直接写存储）
- Agent 写操作留痕（运行记录标注触发来源）

**验收标准：**

- [ ] Claude Desktop / Claude Code 配置 MCP 后可连接
- [ ] Agent 调用 `get_stock_data` 返回行情数据
- [ ] Agent 调用 `evaluate_factor` 触发因子评价任务
- [ ] Agent 调用 `get_job_status` 查询任务进度
- [ ] Agent 触发的操作在运行记录中标注来源为 "agent"

---

- 资产依赖链完整实现
  - 上游变更提示（"依赖的因子已有更新版本"）
  - 信号生成前依赖链全链校验
- DuckDB 表：`signal_runs`、`signal_details`
- API：`/api/signals/generate` + `/api/signals/history`

**验收标准：**

- [ ] 选择 Published 策略 → 生成信号 → 输出信号列表
- [ ] 依赖链中有 Draft 资产时，信号降级为探索性
- [ ] 数据过期时警告并标注
- [ ] 信号历史可查看，包含依赖链快照
- [ ] 导出 CSV/JSON 格式正确

- [ ] 资产生命周期状态在各页面一致展示
- [ ] 探索性/正式结果标识贯穿系统

**Phase 4 整体验收：** 人和 Agent 均可使用完整系统。研究闭环成立。

## 里程碑依赖关系

```
M1.1 ──→ M1.2 ──→ M1.3 ──→ M1.4
                    │
                    ▼
          M2.1 ──→ M2.2 ──→ M2.3
                              │
                              ▼
                    M3.1 ──→ M3.2 ──→ M3.3 ──→ M3.4
                                                 │
                                                 ▼
                                       M4.1 ──→ M4.2 ──→ M4.3
```

- M1.1 无前置依赖
- M1.2 依赖 M1.1（DuckDB、配置、任务框架）
- M1.3 依赖 M1.2（需要股票数据才能做分组）
- M1.4 依赖 M1.2 + M1.3
- M2.1 依赖 M1.2（需要行情数据做指标计算）
- M2.2 依赖 M2.1（因子协议、标签定义、指标库）
- M2.3 依赖 M2.2（因子计算和评价引擎）
